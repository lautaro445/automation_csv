Automation CSV - Quick Start Guide

Requirements:
- Docker installed
- On Windows: Docker Desktop (includes Docker Compose)
- CSV files placed in 'data' folder
- Create a .env file with EMAIL_USER and EMAIL_PASS

Linux / Mac:
1. Open terminal in the project folder
2. Run:
   ./run.sh             # default: simulate emails
   ./run.sh simulate    # simulate emails
   ./run.sh send        # send all emails
   ./run.sh send 5      # send only 5 emails for testing

Windows:
1. Open CMD or PowerShell in the project folder
2. Run:
   run.bat             # default: simulate emails
   run.bat simulate    # simulate emails
   run.bat send        # send all emails
   run.bat send 5      # send only 5 emails for testing

Outputs:
- Files generated in 'output/':
  filtered_emails.csv – emails filtered by domain
  valid_emails.csv    – all valid emails
  invalid_emails.csv  – invalid emails

- Logs stored in 'logs/app.log'

Tips:
- Always run simulate first to check results
- For demos or client tests, use the LIMIT option
